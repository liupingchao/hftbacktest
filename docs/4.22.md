## 2. 已实现功能

### 2.1 回测主线与核心模型

### 2.2 定价与库存控制

已实现并在策略主循环中生效：

- `fair = mid + w_imb*(bid_size-ask_size) + w_spread*spread + w_vol*sigma`
- `reservation = fair - k_inv*position`
- `half_spread = base + k_vol*sigma + k_pos*abs(position) + impact_cost + min(0.05, sigma*0.1)`
- `bid/ask` clamp 边界约束（不穿透最优价）

### 2.3 订单管理（6 条规则）

已实现：

1. 双边各挂 1 单（维护 one-order-per-side）
2. 价格变化 `<1 tick` 不更新
3. 价格变化 `>1 tick` 执行“先撤后挂”
4. 仓位超限仅挂减仓侧
5. API 间隔 `<20ms` 跳过本次更新
6. 延迟 `>5ms` 不发单（`latency_guard`）

### 2.4 延迟与限流仿真

已实现：

- `IntpOrderLatency` 接入（支持由实盘审计生成）
- 尖刺注入：1% 概率，entry/resp 都可注入 `8~10ms`
- `entry > 5ms` 拒单并打 `reject_reason=latency_guard`
- 令牌桶限流 + API 间隔门控（审计可追踪）

## 4. 回测使用方法（当前可直接执行）

### 4.1 Mac：环境准备

```bash
conda create -y -n hftbacktest python=3.12 pip
conda activate hftbacktest
pip install hftbacktest==2.4.4
```

### 4.2 配置文件准备

```bash
cd /Users/liu/Documents/hftbacktest/examples/binance_tick_mm
cp config.example.toml config.toml
```

需要修改的关键项：

- `paths.mac_tardis_dir`（支持 `~/document/tardis` 与 `~/Documents/tardis`）
- `latency.order_latency_npz`
- `data.start_day / data.end_day`
- `paths.output_root`

### 4.3 （可选）由实盘审计生成 latency npz

```bash
python latency_from_audit.py \
  --audit-csv /path/to/audit_live.csv \
  --output-npz /path/to/live_order_latency.npz \
  --output-stats /path/to/live_order_latency_stats.json \
  --entry-ms-min 1.2 --entry-ms-max 2.8 \
  --resp-ms-min 1.0 --resp-ms-max 2.2 \
  --spike-prob 0.01 --spike-ms-min 8.0 --spike-ms-max 10.0
```

### 4.4 Mac：5 分钟回测 + 图输出

```bash
python run_env_test.py \
  --config ./config.toml \
  --target mac \
  --day 2025-01-01 \
  --plot
```

## 6. 注意事项

- `strict_timestamps=true` 时，若源数据非严格递增会 fail-fast，并输出 `strict_report_*.json`。
- returns 图当前是 MTM 近似（基于 inventory * dmid 的累计），用于快速诊断策略轨迹；如需交易费/滑点精细收益分解，可后续扩展。
- 当前已优先保证 BTC 单品种可复现流程；多品种与长周期可在此基础上扩展。
